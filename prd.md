# MBTI 성격 유형 테스트 웹사이트 PRD

## 1. 프로젝트 개요

### 1.1 서비스명
**"MBTI 찐테스트"** (가칭)

### 1.2 한 줄 소개
MZ세대를 위한 재미있고 공유하고 싶은 MBTI 성격 유형 테스트 서비스

### 1.3 서비스 비전 및 미션
**비전**: "가장 재미있게 나를 알아가는 MBTI 테스트"

**미션**:
- MZ세대가 공감할 수 있는 유머러스한 질문과 결과로 자기 이해를 돕는다
- 친구들과 공유하고 싶은 콘텐츠로 바이럴 확산을 유도한다
- 심리학적 깊이보다는 재미와 공감을 우선시하여 진입장벽을 낮춘다

### 1.4 핵심 가치 제안 (Value Proposition)

**문제 인식**:
- 기존 MBTI 테스트는 질문이 너무 많고 지루함 (50+ 문항)
- 딱딱하고 전문적인 용어로 결과가 제시되어 재미가 없음
- 결과를 친구들과 공유하기 어려운 UI/UX
- 모바일 최적화가 안 되어 있어 이탈률이 높음

**우리의 솔루션**:
- 단 12개 질문으로 빠르게 완료 (3분 이내)
- MZ세대 감성의 유머러스한 질문과 밈(meme) 활용
- 결과를 이미지로 바로 저장하여 인스타/카톡에 공유 가능
- 모바일 퍼스트 디자인으로 어디서든 편하게 테스트

### 1.5 경쟁 서비스 분석 및 차별화 포인트

| 요소 | 16Personalities | MBTI 공식 테스트 | **우리 서비스** |
|------|-----------------|------------------|-----------------|
| 질문 수 | 60+ 문항 | 93 문항 | **12 문항** |
| 소요 시간 | 12분+ | 20분+ | **3분 이내** |
| 톤앤매너 | 전문적 | 학술적 | **친근하고 재미있음** |
| 모바일 최적화 | 보통 | 낮음 | **높음 (모바일 퍼스트)** |
| 공유 기능 | 링크만 | 없음 | **이미지 저장/공유** |
| 회원가입 | 선택적 | 필수 | **불필요** |
| 가격 | 무료 | 유료 | **무료** |

**핵심 차별화 포인트**:
1. **초단기 완성형**: 12문항으로 3분 안에 끝내는 스피드감
2. **MZ세대 언어**: "집에 가고 싶다 vs 놀고 싶다" 같은 공감 가는 질문
3. **공유 최적화**: 결과를 예쁜 이미지로 저장하여 SNS 공유 유도
4. **제로 진입장벽**: 회원가입 없이 바로 시작

## 2. 타겟 사용자 분석

### 2.1 주요 타겟 (Primary Target)

**인구통계학적 특성**:
- 연령: 15세 ~ 25세 (고등학생, 대학생, 사회초년생)
- 성별: 전 성별 (여성 비율 약 65% 예상)
- 지역: 전국 (모바일 중심 접속)
- 디바이스: 모바일 80%, 데스크톱 20%

**행동 패턴**:
- 하루 평균 3시간 이상 SNS 사용
- 인스타그램 스토리, 카카오톡으로 일상 공유
- 짧고 재미있는 콘텐츠 선호 (틱톡, 릴스, 숏츠)
- 친구들과 함께 즐길 수 있는 콘텐츠에 반응
- MBTI에 대한 관심이 높음 (일상 대화 주제)

**니즈**:
- 심심할 때 3분 안에 할 수 있는 재미있는 콘텐츠
- 친구들과 비교하고 공유할 수 있는 소재
- 나에 대해 알아가는 가벼운 자기 탐색
- 예쁘고 공유하기 좋은 결과물

### 2.2 보조 타겟 (Secondary Target)

**26세 ~ 35세 직장인**:
- MBTI를 팀 빌딩이나 자기계발 목적으로 활용
- 점심시간, 퇴근 후 여가 시간에 접속
- 결과의 신뢰도보다는 재미와 공감 중심

4. 결과를 이미지로 저장해서 인스타 스토리에 업로드
5. 친구들 반응 보면서 서로 MBTI 비교

## 3. 핵심 기능 목록

### 3.1 기능 우선순위

#### Must Have (MVP 필수 기능)
1. **테스트 시스템**
   - 12개 질문 기반 MBTI 판별
   - 프로그레스 바 표시
   - 이전 질문 돌아가기 기능

2. **결과 표시**
   - 16가지 MBTI 유형별 결과 페이지

   - 유머러스한 설명과 이미지

3. **공유 기능**
   - 결과를 이미지로 저장 (Canvas API 활용)

4. **통계 기능**
   - 전체 테스트 참여자 수
   - 16가지 유형별 비율 (차트)
   - 가장 많은 유형 표시

5. **랜딩 페이지**
   - 서비스 소개
   - 시작하기 버튼
   - 간단한 통계 미리보기

5. **상세 통계** (성별, 연령대별 분포)

### 3.2 기능별 상세 설명 (User Story 형식)

#### 기능 1: 테스트 시스템

**사용자 스토리**:
- 사용자로서, 12개의 재미있는 질문에 답하고 싶다, 내 MBTI를 알기 위해
- 사용자로서, 현재 몇 번째 질문인지 알고 싶다, 얼마나 남았는지 확인하기 위해
- 사용자로서, 이전 질문으로 돌아가고 싶다, 답변을 수정하기 위해

**상세 요구사항**:
- 총 12개 질문 (E/I 3문항, S/N 3문항, T/F 3문항, J/P 3문항)
- 각 질문은 2지선다 또는 5점 척도 (강하게 A / 약하게 A / 중립 / 약하게 B / 강하게 B)
- 상단에 프로그레스 바 (예: "3 / 12")
- 하단에 "이전" 버튼 (1번 질문에서는 비활성화)
- 질문마다 재미있는 이미지나 GIF 삽입
- 답변 선택 시 부드러운 애니메이션 효과

**질문 예시**:
1. (E/I) 주말에 친구들이 놀자고 할 때
   - 강하게 동의: "좋아! 어디 가?" (E)
   - 약하게 동의: "음... 뭐 하는데?" (E)
   - 중립: "상황 봐서"
   - 약하게 반대: "피곤한데..." (I)
   - 강하게 반대: "집이 최고야" (I)

2. (S/N) 친구가 "너 내일 뭐 해?" 물어보면
   - A: "아침 10시에 일어나서, 점심 먹고, 카페 가고..." (S - 구체적)
   - B: "그냥 여유롭게 보낼 거야" (N - 추상적)

#### 기능 2: 결과 표시

**사용자 스토리**:
- 사용자로서, 내 MBTI 유형과 특징을 보고 싶다, 나를 이해하기 위해
- 사용자로서, 재미있고 공감되는 설명을 보고 싶다, 친구들과 공유하기 위해
- 사용자로서, 내 유형이 전체의 몇 %인지 알고 싶다, 희귀도를 확인하기 위해

**상세 요구사항**:
- 결과 페이지 구성 요소:
  1. **헤더**: "당신은 [MBTI 유형]입니다!" + 대표 별명 (예: ENFP - 스파클 마스터)
  2. **대표 이미지**: 유형별 특징을 나타내는 재미있는 일러스트/밈
  3. **한 줄 요약**: "친구들의 비타민, 에너지 발전소"
  4. **특징**: 3-5개 핵심 특징 (이모지 활용)
  5. **강점**: 3개 (긍정적 표현)
  6. **약점**: 3개 (유머러스하게 표현)
  7. **추천 직업**: 5-7개
  8. **연예인 예시**: 같은 MBTI로 알려진 연예인 3-5명
  9. **비율**: "전체의 X% | 상위 X번째로 많은 유형"
  10. **CTA 버튼들**: 다시하기, 공유하기, 통계 보기

**ENFP 결과 예시**:
```
당신은 ENFP입니다!
"스파클 마스터 - 친구들의 비타민"

[재미있는 ENFP 일러스트]

한 줄 요약
"아이디어는 넘치는데 실행은... 음... 내일 할게요!"

당신의 특징
- 번뜩이는 아이디어 뱅크
- 창의력 MAX, 집중력은... 글쎄?
- 사교성 99% (배터리 1%)
- 감정 기복이 롤러코스터
- 시작은 열정적, 끝은 미지수

강점
• 친화력 甲 - 낯선 사람과도 금방 친해짐
• 창의적 문제 해결 능력
• 긍정 에너지로 분위기 메이커

약점
• 계획은 세우는데 실행은... (3일 법칙의 달인)
• 집중력 지속 시간 = 금붕어 수준
• 혼자 있으면 외로운데 사람 만나면 피곤함

추천 직업
기획자, 마케터, 크리에이터, 상담가, 이벤트 플래너,
방송인, 디자이너

같은 MBTI 연예인
아이유, 뷔(BTS), 박나래, 유재석(?), 지드래곤

전체의 8.1% | 5번째로 많은 유형
```

#### 기능 3: 공유 기능

**사용자 스토리**:
- 사용자로서, 결과를 이미지로 저장하고 싶다, 인스타그램에 올리기 위해
- 사용자로서, 링크를 복사하고 싶다, 친구들에게 카톡으로 보내기 위해
- 사용자로서, 원클릭으로 SNS에 공유하고 싶다, 편리하게 자랑하기 위해

**상세 요구사항**:

1. **이미지 저장 기능**:
   - 결과 페이지를 Canvas API로 이미지화
   - 크기: 1080x1920 (인스타그램 스토리 최적 비율)
   - 배경: 그라데이션 또는 패턴
   - 포함 요소: MBTI 유형, 별명, 한 줄 요약, 대표 특징 3개, 하단에 서비스 로고/URL
   - "이미지로 저장" 버튼 클릭 시 다운로드
   - 파일명: `MBTI-결과-{유형}-{날짜}.png`

2. **링크 복사 기능**:
   - "링크 복사" 버튼 클릭
   - URL에 결과 유형 포함 (예: `https://mbti-test.com/result/ENFP`)
   - 클립보드에 복사 후 토스트 메시지 "링크가 복사되었습니다!"

3. **SNS 공유 버튼**:
   - 카카오톡: Kakao SDK 활용, 커스텀 템플릿
   - 페이스북: Facebook SDK
   - 트위터: Twitter Web Intent
   - URL 복사: 네이티브 클립보드 API
   - 각 버튼에 아이콘 + 텍스트

4. **공유 시 메타데이터**:
   - OG 태그 설정으로 링크 미리보기 최적화
   - 제목: "나는 {MBTI 유형}! 너는?"
   - 설명: "3분 만에 알아보는 재미있는 MBTI 테스트"
   - 이미지: 유형별 대표 이미지

#### 기능 4: 통계 기능

**사용자 스토리**:
- 사용자로서, 다른 사람들은 어떤 유형이 많은지 보고 싶다, 나와 비교하기 위해
- 사용자로서, 지금까지 몇 명이 테스트했는지 알고 싶다, 인기를 확인하기 위해
- 사용자로서, 내 유형의 희귀도를 알고 싶다, 특별함을 느끼기 위해

**상세 요구사항**:

1. **통계 페이지 구성**:
   - 상단: 전체 참여자 수 (카운터 애니메이션)
   - 중단: 16가지 유형 분포 차트 (도넛 차트 또는 바 차트)
   - 하단: TOP 3 / BOTTOM 3 유형

2. **데이터 표시 방식**:
   - 각 유형별 퍼센트 + 실제 참여자 수
   - 시각적 차트 (Chart.js 또는 Recharts 활용)
   - 내 유형 하이라이트 (다른 색상으로 강조)

3. **실시간 업데이트**:
   - 결과 제출 시 통계 DB에 카운트 증가
   - 통계 페이지 접속 시 최신 데이터 fetch
   - 캐싱 전략: 1분 단위로 갱신

4. **추가 정보**:
   - "가장 많은 유형: ENFP (15.2%)"
   - "가장 적은 유형: INFJ (1.8%)"
   - "당신의 ENFP는 상위 5번째입니다"

## 4. 사용자 플로우

### 4.1 핵심 사용자 여정 (User Journey Map)

```
[접속] → [랜딩 페이지] → [테스트 시작] → [질문 응답] → [결과 확인] → [공유] → [재방문/이탈]
```

**단계별 감정 곡선**:
- 접속: 중립 - "뭐지? 한번 해볼까?"
- 랜딩 페이지: 흥미 - "오 간단하네, 해보자!"
- 질문 1-4: 재미 - "ㅋㅋ 이거 나네"
- 질문 5-8: 몰입 - "진짜 공감되네"
- 질문 9-12: 기대 - "결과 궁금하다!"
- 결과 로딩: 기대 최고조 - "두근두근"
- 결과 확인: 만족 - "완전 나잖아!"
- 공유: 흥분 - "친구들한테 보여줘야지!"

### 4.2 주요 시나리오별 플로우

#### 시나리오 1: 최초 방문 사용자 (모바일)

```
1. 친구가 공유한 링크 클릭 (카카오톡)
   ↓
2. 랜딩 페이지 로드 (2초 이내)
   - 히어로 섹션 표시
   - "테스트 시작하기" 버튼 보임
   ↓
3. "테스트 시작하기" 버튼 클릭
   ↓
4. 질문 1/12 페이지로 이동
   - 프로그레스 바: "1 / 12"
   - 질문 및 2개 선택지 표시
   - 하단: "이전" (비활성화), "다음" (답변 선택 후 활성화)
   ↓
5. 답변 선택
   - 버튼 클릭 시 체크 표시 + 색상 변경
   - "다음" 버튼 활성화
   ↓
6. "다음" 버튼 클릭 또는 답변 자동 진행 (선택 시 0.5초 후 자동 넘어가기)
   ↓
7. 질문 2/12 → 질문 3/12 → ... → 질문 12/12
   - 각 질문마다 동일한 플로우
   - "이전" 버튼으로 이전 질문 수정 가능
   ↓
8. 질문 12/12 답변 후
   - 로딩 애니메이션 (2초) "당신의 MBTI를 분석 중..."
   ↓
9. 결과 페이지 로드
   - 스크롤 가능한 긴 페이지
   - 상단: MBTI 유형 + 별명
   - 중단: 특징, 강점, 약점, 추천 직업
   - 하단: 공유 버튼, 다시하기 버튼
   ↓
10. 결과 확인 후 "이미지로 저장" 클릭
   - Canvas로 이미지 생성 (1초)
   - 이미지 다운로드
   - 토스트 메시지: "이미지가 저장되었습니다!"
   ↓
11. 인스타그램 앱으로 이동
   - 스토리에 이미지 업로드
   ↓
12. (선택) 다시 사이트로 돌아와서 "다시하기" 또는 이탈
```

#### 시나리오 2: 통계 확인

```
1. 결과 페이지 하단 "전체 통계 보기" 버튼 클릭
   ↓
2. 통계 페이지로 이동
   - 상단: "지금까지 XXX명이 참여했어요!"
   - 중단: 16가지 유형 분포 차트
   - 내 유형 하이라이트 표시
   ↓
3. 차트 인터랙션
   - 각 유형에 마우스 오버 시 상세 정보 툴팁
   - 모바일: 탭하면 상세 정보
   ↓
4. "내 결과로 돌아가기" 버튼 클릭
   ↓
5. 결과 페이지로 복귀
```

#### 시나리오 3: 링크를 통한 재방문

```
1. 이전에 받은 결과 URL 클릭 (예: /result/ENFP)
   ↓
2. 해당 MBTI 결과 페이지로 바로 이동
   - 테스트 다시 하지 않고 결과만 확인 가능
   ↓
3. "내 결과도 확인하기" 버튼
   ↓
4. 테스트 시작
```

### 4.3 온보딩 프로세스

**별도 온보딩 없음** (제로 프릭션):
- 회원가입 불필요
- 튜토리얼 불필요 (직관적인 UI)
- 랜딩 페이지가 곧 온보딩
- "테스트 시작하기" 클릭 → 바로 첫 질문

**첫 질문 페이지에서 간단한 안내**:
- 상단에 작은 툴팁: "각 질문에 솔직하게 답변해주세요 (12문항)"
- 1초 후 자동으로 사라지거나 X 버튼으로 닫기

### 4.4 예외 상황 처리 플로우

#### 예외 1: 중간에 페이지 이탈 후 재접속

**문제**: 사용자가 질문 5번까지 답하다가 페이지를 닫음

**해결**:
- LocalStorage에 진행 상황 자동 저장
- 재접속 시 모달 표시: "이전에 5번까지 답변하셨어요. 이어서 할까요?"
  - [이어서 하기] → 질문 6번으로 이동
  - [처음부터 하기] → LocalStorage 초기화, 질문 1번으로 이동

#### 예외 2: 답변을 선택하지 않고 "다음" 클릭

**문제**: 사용자가 실수로 답변 없이 다음으로 넘어가려 함

**해결**:
- "다음" 버튼 기본 비활성화
- 답변 선택 시에만 활성화
- (대안) 답변 선택 시 0.5초 후 자동으로 다음 질문으로 이동

#### 예외 3: 결과 이미지 저장 실패

**문제**: Canvas API 지원 안 되는 브라우저 또는 권한 오류

**해결**:
- Try-Catch로 에러 핸들링
- 에러 발생 시 대체 방법 안내:
  - "이미지 저장이 지원되지 않는 브라우저입니다."
  - "스크린샷을 이용해주세요" + 스크린샷 가이드 표시
  - 또는 서버에서 이미지 생성 후 다운로드 링크 제공

#### 예외 4: 네트워크 오류

**문제**: 결과 제출 시 네트워크 끊김

**해결**:
- 로딩 중 스피너 표시
- 5초 타임아웃 후 에러 메시지:
  - "네트워크 오류가 발생했습니다."
  - [다시 시도] 버튼
- 답변 데이터는 LocalStorage에 보관되어 있어 다시 제출 가능

#### 예외 5: 통계 데이터 로드 실패

**문제**: 통계 API 응답 없음

**해결**:
- 스켈레톤 UI 표시
- 3초 후 재시도
- 실패 시 기본 메시지: "통계를 불러올 수 없습니다. 잠시 후 다시 시도해주세요."
- 결과 페이지는 정상 표시 (통계는 부가 기능)

### 5.6 반응형 디자인 고려사항

#### Breakpoints
- **Mobile**: 0 ~ 767px (기본)
- **Tablet**: 768px ~ 1023px
- **Desktop**: 1024px +

#### Mobile (기본)
- 컨테이너 너비: 100%
- 패딩: 16px (좌우)
- 1단 레이아웃

#### Tablet
- 컨테이너 너비: 720px (중앙 정렬)
- 패딩: 24px (좌우)
- 질문 선택지: 가로 2개씩

#### Desktop
- 컨테이너 너비: 1024px (중앙 정렬)
- 패딩: 48px (좌우)
- 결과 페이지: 2단 레이아웃

### 5.7 인터랙션 디자인

#### 마이크로 인터랙션

1. **버튼 호버**: Scale 1 → 1.05, Duration 150ms
2. **버튼 클릭**: Scale 1.05 → 0.95 → 1, Ripple 효과
3. **질문 답변 선택**: 배경 색상 변경 + 체크 아이콘
4. **페이지 전환**: Fade in + Slide up, Duration 400ms
5. **프로그레스 바**: 부드럽게 증가, Duration 500ms
6. **카운터 애니메이션**: 0부터 증가, Duration 2000ms

### 5.8 접근성 (A11y) 고려사항

1. **색상 대비**: WCAG AA 기준 준수 (4.5:1 이상)
2. **키보드 네비게이션**: Tab으로 모든 요소 접근 가능
3. **스크린 리더**: 시맨틱 태그 + ARIA 레이블
4. **동작 감소**: prefers-reduced-motion 지원

### 7.3 데이터베이스 스키마 (PostgreSQL)

```sql
-- MBTIType 테이블 (정적 데이터)
CREATE TABLE mbti_types (
  type VARCHAR(4) PRIMARY KEY,
  nickname VARCHAR(100) NOT NULL,
  summary TEXT NOT NULL,
  characteristics JSONB NOT NULL,
  strengths JSONB NOT NULL,
  weaknesses JSONB NOT NULL,
  recommended_jobs JSONB NOT NULL,
  celebrities JSONB NOT NULL,
  image_url TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Question 테이블 (정적 데이터)
CREATE TABLE questions (
  id SERIAL PRIMARY KEY,
  order_num INTEGER NOT NULL UNIQUE,
  dimension VARCHAR(1) NOT NULL CHECK (dimension IN ('E', 'I', 'S', 'N', 'T', 'F', 'J', 'P')),
  text TEXT NOT NULL,
  option_a TEXT NOT NULL,
  option_b TEXT NOT NULL,
  image_url TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- TestResult 테이블 (사용자 제출 결과)
CREATE TABLE test_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  mbti_type VARCHAR(4) REFERENCES mbti_types(type),
  answers JSONB NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  user_agent TEXT,
  ip_hash VARCHAR(64)
);

CREATE INDEX idx_test_results_mbti_type ON test_results(mbti_type);
CREATE INDEX idx_test_results_created_at ON test_results(created_at);

-- Statistics 테이블 (실시간 통계)
CREATE TABLE statistics (
  id SERIAL PRIMARY KEY,
  mbti_type VARCHAR(4) UNIQUE REFERENCES mbti_types(type),
  count INTEGER DEFAULT 0,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 트리거: TestResult INSERT 시 Statistics 자동 업데이트
CREATE OR REPLACE FUNCTION update_statistics()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO statistics (mbti_type, count, updated_at)
  VALUES (NEW.mbti_type, 1, CURRENT_TIMESTAMP)
  ON CONFLICT (mbti_type)
  DO UPDATE SET
    count = statistics.count + 1,
    updated_at = CURRENT_TIMESTAMP;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_statistics
AFTER INSERT ON test_results
FOR EACH ROW
EXECUTE FUNCTION update_statistics();
```
